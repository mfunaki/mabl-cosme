# コードレビュープロンプト

## 目的
指定されたコードの品質をレビューし、改善点を提案する。

## レビュー観点

### 1. コード品質
- [ ] 可読性: 変数名、関数名は明確か
- [ ] 保守性: 単一責任の原則を守っているか
- [ ] 重複: DRY 原則に反していないか

### 2. パフォーマンス
- [ ] 不要な再レンダリングはないか
- [ ] メモ化が必要な箇所はないか
- [ ] N+1 問題はないか

### 3. セキュリティ
- [ ] XSS 脆弱性はないか
- [ ] 機密情報がハードコードされていないか
- [ ] 入力値のバリデーションは適切か

### 4. エラーハンドリング
- [ ] try-catch が適切に使われているか
- [ ] ユーザーへのエラー表示は適切か
- [ ] ログ出力は十分か

### 5. テスタビリティ
- [ ] テストしやすい構造か
- [ ] data-testid が適切に付与されているか

## 出力形式

```markdown
## レビュー結果

### 良い点
- ...

### 改善提案
1. **[重要度: 高/中/低]** 問題の説明
   - 現状: ...
   - 提案: ...
   - 理由: ...

### コード例（改善後）
```

## 使用例

```
@.claude/prompts/code-review.md を使って src/App.tsx をレビューして
```

# テストコード生成プロンプト

## 目的
指定されたコンポーネントまたは関数に対するテストコードを生成する。

## 指示

以下の手順でテストを作成してください：

1. **対象コードを分析**
   - 入力パラメータと戻り値の型を確認
   - 副作用（API呼び出し、状態変更など）を特定
   - エッジケースを洗い出す

2. **テストケースを設計**
   - 正常系: 期待通りの入力で期待通りの出力
   - 異常系: 不正な入力、エラーハンドリング
   - 境界値: 最小値、最大値、空配列など

3. **テストコードを作成**
   - describe/it 構造で整理
   - AAA パターン（Arrange, Act, Assert）を使用
   - モックが必要な場合は適切に設定

## 制約

- TDD の原則に従い、まずテストを作成
- 実装コードは変更しない
- data-testid を活用してセレクタを安定させる

## 出力形式

```typescript
describe('対象の名前', () => {
  it('should ...', () => {
    // Arrange
    // Act
    // Assert
  })
})
```

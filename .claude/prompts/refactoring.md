# リファクタリングプロンプト

## 目的
既存コードの動作を変えずに、構造や可読性を改善する。

## リファクタリングの種類

### 1. 関数の抽出
長い関数を小さな関数に分割する。

### 2. コンポーネントの分割
大きなコンポーネントを再利用可能な小さなコンポーネントに分割する。

### 3. 型の改善
any 型を具体的な型に置き換える。

### 4. 命名の改善
より明確で一貫性のある命名に変更する。

### 5. 重複の除去
共通処理をユーティリティ関数やカスタムフックに抽出する。

## 制約

- **動作を変えない**: リファクタリング前後で同じ動作を保証
- **段階的に進める**: 一度に大きな変更をしない
- **テストで確認**: 各ステップ後にテストを実行

## 手順

1. 対象コードの現状を分析
2. リファクタリングの計画を立てる
3. 変更を小さなステップに分割
4. 各ステップ後に動作確認
5. 完了後に全体をレビュー

## 出力形式

```markdown
## リファクタリング計画

### 現状の問題
- ...

### 改善後のゴール
- ...

### ステップ
1. ...
2. ...

### 変更後のコード
```

## 使用例

